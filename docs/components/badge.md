# Badge

Badges communicate information, status, counts, or categorization.

## When to Use

Use a badge when:

* Indicating a status.
* Displaying a count or notification.
* Categorizing or labeling an item.
* Providing additional contextual information.

Avoid using badges for:

* Actions.
* Navigation.
* Long descriptions.
* Interactive controls.

## Anatomy

A badge contains:

* Container
* Text

## Variants

### Positive

Use for positive statuses or successful outcomes.

Examples:

* Approved
* Success
* Active

### Negative

Use for negative statuses or unsuccessful outcomes.

Examples:

* Rejected
* Failed
* Error

Rules:

* May be used as a counter badge.

### Warning

Use for warnings, pending states, or conditions requiring attention.

Examples:

* Pending
* Waiting for Approval
* In Progress

### Disable

Use for inactive or unavailable states.

Examples:

* Inactive
* Disabled
* Unapprovable

### Info

Use for neutral information or normal states.

Examples:

* Headquarter
* Time In
* Time Out

## Content Types

### Status Badge

Use to communicate the current state of an item.

Examples:

* Active
* Pending
* Rejected

### Counter Badge

Use to display counts or notifications.

Examples:

* 1
* 8
* 99+

Rules:

* Keep values concise.
* Use abbreviated formats when space is limited.

### Label Badge

Use to categorize or classify an item.

Examples:

* Headquarter
* Payroll
* Full Time

## Placement

Use badges close to the content they describe.

Common placements:

* Tables
* Cards
* Detail pages
* Lists
* Navigation items

## Specification Notes

* Badges are non-interactive.
* Badges use a pill shape.
* Badge labels use Title Case.
* Keep badge text concise whenever possible.
