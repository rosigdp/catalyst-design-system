# Text Action

Text actions trigger lightweight actions without the visual emphasis of a button.

## When to Use

Use a text action when:

* The action is contextual to a specific piece of content or component.
* The action is displayed inline with content.
* The action does not require strong visual emphasis.
* The action does not have any representative icon.
* The action label is dynamic or user-generated.

Avoid using text actions for:

* Primary page actions.
* Final confirmation actions.
* Actions that require strong emphasis.

## Variants

### Link

Use for navigational or informational actions.

Examples:

* Learn More linking to Help Center
* Employee name linking to the employee profile
* Menu name linking to related setup

### Action

Use for lightweight actions within a component.

Examples:

* Check All
* View Details
* Show More

Rules:

* The action should affect only the current component or content area.

### Destructive

Use for lightweight destructive actions within a component.

Examples:

* Uncheck All
* Clear Filters
* Remove Selection

Rules:

* Use only when the action may alter, remove, reset, or invalidate data within the current component.

## Grouping

Rules:

* Display text actions inline.
* When action and destructive text actions appear together, place the action on the right.
* Text actions and icon actions may appear together in the same action group.
* Do not mix text actions with buttons within the same action group.

## Specification Notes

* Text actions contain text only.
* Text actions do not have a disabled state.
* Text actions use Title Case.
* Action and destructive labels should use verbs whenever possible.
* Use icon actions when space is limited and the icon meaning is widely understood.
* Use buttons when stronger visual emphasis is required.
